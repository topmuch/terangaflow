## Architecture Decisions

1. **Zustand over next-intl routing**: Since this is a single-page app (only `/` route), full next-intl middleware was unnecessary. Zustand provides simpler state management with localStorage persistence.

2. **Locale detection chain**: localStorage → `navigator.language` → fallback `'fr'`. French is the default since the app targets Senegal.

3. **Parameter interpolation**: `t('display.inMin', { min: 5 })` → "Dans 5 min". Uses `{param}` syntax with regex replacement.

4. **Wolof translations**: Used authentic Wolof vocabulary for common transport terms where they are widely understood. Technical terms (API, Dashboard, QR, Analytics) kept in French as is standard practice in Senegal.

5. **No next.config changes**: No middleware or routing configuration needed. Purely client-side i18n.

## Integration Guide

To use the i18n system in any component:

```tsx
import { useT } from '@/components/i18n/use-t'
import { LanguageSwitcher } from '@/components/i18n/language-switcher'

function MyComponent() {
  const { t } = useT()
  return (
    <div>
      <LanguageSwitcher />
      <h1>{t('display.departures')}</h1>
      <p>{t('display.inMin', { min: 5 })}</p>
    </div>
  )
}
```
